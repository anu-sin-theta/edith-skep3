It does not currently have native decoding for ERC-721 or ERC-1155 NFT events. The AI might catch it from raw logs, but deterministic NFT drain warnings are an active limitation (also noted in the Roadmap).
AI Context Window Truncation: To save memory and fit into local Ollama models (like qwen3:4b-instruct), the parser aggressively truncates data:
Contract code injected into the AI prompt is capped at 25,000 characters.
Raw trace calldata inputs are sliced at 512 characters. Highly complex, nested, or massively obfuscated malicious contracts might hide their payloads outside of these windows, causing the AI to evaluate an incomplete picture.
No Native Reentrancy Loop Detection: The current execution tracer does not natively calculate recursive call depth or flag reentrancy loops automatically. Relying purely on the 4B parameter AI to mentally connect recursive depth limits its ability to spot advanced reentrancy exploits.
False Positive Risks on Standard Proxies: The Layer 2 pattern matcher indiscriminately flags DELEGATECALL as a "suspicious opcode". Because almost all modern DeFi protocols (like AAVE or USDC) use upgradeable proxy patterns, the system leans entirely on the local LLM's reasoning to distinguish between a legitimate proxy and a drainer proxy. A "hallucination" by the LLM here leads to immediate false positives.
⚙️ Simulation Limitations (Anvil Sandbox)
Isolated "Point-in-Time" Context: The Anvil EVM fork takes a snapshot at the exact current block and runs your single transaction in a completely isolated parallel universe. It cannot foresee front-running (MEV bot interference), sandwich attacks, or other state changes that might occur earlier in the exact same block on the real mainnet before your transaction is actually mined.
Single-Transaction Myopia: It evaluates one transaction at a time. It cannot foresee delayed-execution attacks or multi-step exploits. For example, if an attacker requires you to sign a harmless-looking transaction first and a separate malicious transaction an hour later, the simulator views the first transaction in isolation and will rate it as SAFE.
Heavy RPC Dependency & Rate Limiting: Even though simulation happens locally in RAM, Anvil is entirely reliant on pulling state (eth_getCode, eth_getStorageAt) via free public RPCs (ethereum.publicnode.com or llamarpc). If the public RPC rate-limits the node, or goes out of sync, the simulation will either fail or return completely inaccurate state differences.
Single-Chain Limitation: Currently, the system is strictly tailored for the Ethereum Mainnet. Attempting to simulate transactions meant for Layer-2s (Arbitrum, Base, Optimism) or alternative Layer-1s (BSC, Polygon) will fail contextually because the fork targets Ethereum RPCs by default.
Off-Chain Blindspots: If a dApp relies heavily on an off-chain relayer network, centralized backend state, or EIP-712 off-chain signatures to execute logic, the local Anvil fork cannot simulate or verify those external off-chain conditions.
